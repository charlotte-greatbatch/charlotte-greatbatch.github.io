Display inline vs. inline-block?

If you're anything like me, who had never heard of that term before I saw this question, Welcome! Here I am going to explain the difference between displaying inline vs displaying inline-block.

Firstly, what is inline and what is inline-block?

We need to define what both inline and block are.

Inline takes elements and display them side by side. They take up space as needed, and the width and height cannot be set. Examples of inline elements are a, span, img and button.

Block-line element takes 100% of the width of the page by default. It'll start each element on a new line, the width and height CAN be set, and examples are div, p, h1, ul and foooter.

So, after some small techincal talk, let's break it down easier to Inline vs. Inline-block

Inline:

  • Will display elements like links, images, buttons, etc side by side
  • The width and height of those elements cannot be set
  • Top and bottom margins (spacing between lines) are not respected

Inline-block:

  • Will display elements side by side
  • Width and height can be set
  • Respects the elements around it, including top and bottom margins (spacing)

In conclusion

Inline-block gives you the flexibility for the customisation width/height of an element and it respects the top and bottom margins.
